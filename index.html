二分查找

二分查找也叫折半查找，常见于在一个单调序列中查找找某一个数，但二分查找并不仅限于单调序列；如果能够找到一个性质，能够把序列一分为二，一半满足这个性质一半不满足，既可以用二分来查找这个性质的边界，即点A或点B。

二分查找点A和点B会有点不同
当你想要查找点A，即红色部分是满足性质的；

  1. 如果check(mid)为false；这时mid的位置不可能为答案，所以需要把r更新为mid-1，l不变；
  2. 如果check(mid)为true；这是mid的位置有可能是你想查找的数，所以把l更新为mid，r不变；

线性筛法

线性筛法保证了每一个合数都只会被他的最小质因子筛去。

  • 当 i % p [ j ] == 0 时
    p [ j ] 一定是 i 的最小质因子，p [ j ]一定也是 p [ j ] * i 的最小质因子。
    此时如果没有break，j + 1 则会筛掉 p [ j + 1 ] * i 。
    而由于i的最小质因子是 p [ j ]，所以 p [ j + 1 ] * i 的最小质因子也应该时 p [ j ]，则发生了 p [ j + 1 ] * i 被大于他的最小质因子筛掉的情况。

  • 当 i % p [ j ] != 0 时
    p [ j ] 一定小于i的所有质因子，p [ j ] 也一定是 p [ j ] * i 的最小质因子。

组合数的常见求法

發表於 2019-11-22 更新於 2020-02-25 分類於 数学知识

常用公式

定义式:

递推式：$C^b_a=C^b_{a-1}+C^{b-1}_{a-1}$

卢卡斯定理：
$C^b_a \equiv C^{bmodp}_{amodp} \cdot C^{b/p}_{a/p} (modp)$

$C(a,b)modp=C(a/p,b/p)*C(amodp,bmodp)modp$

方法一（预处理出所有组合数）

当组合数个数不多时，我们可以利用递推式将所有可能的组合数求出来。

方法二（利用乘法逆元求组合数）

当组合数个数较多时且需做取模运算时，可利用定义式求组合数
由于不能对除法取模，所以需要将求出分母的逆元

注意：只有当模上一个质数是才可用快速幂求逆元(费马小定理)，否则只能用扩展欧几里得定理求逆元。

如果询问个数多时可先预处理出每个数的阶乘以及他的逆元
$fact[i] = i! mod p$
$infact[i] = (i!)^{-1} mod p$

$C^b_a = \frac{a!}{b!\cdot(a-b)!} = fact[a]\cdot infact[b-a] \cdot infact[b]$

方法三（卢卡斯定理求解组合数）

遇到大组合数取模(b<=a<1e18)时
$C(a,b)modp=C(a/p,b/p)*C(amodp,bmodp)modp$

方法四（高精度求组合数）

分解C(a,b)中所有质因子，利用高精度乘法求解

利用数组实现单链表

若用结构体实现单链表，在创建结点的时候会耗费大量时间。因此，在算法题中，可以利用数组来实现单链表优化时间。

模板题

实现一个单链表，链表初始为空，支持三种操作：

  1. 向链表头插入一个数；
  2. 删除第k个插入的数后面的数；
  3. 在第k个插入的数后插入一个数

利用数组实现双链表

双链表的操作与单链表类似，只是多了一个数组用于存储前驱结点的指针。

模板题

实现一个双链表，双链表初始为空，支持5种操作：

  1. 在最左侧插入一个数；
  2. 在最右侧插入一个数；
  3. 将第k个插入的数删除；
  4. 在第k个插入的数左侧插入一个数；
  5. 在第k个插入的数右侧插入一个数

数组实现字典树(Trie)

什么是字典树？

Trie,又叫前缀树。通常用来存储字符串，树中的某个结点的子节点都有相同的前缀。根节点对应空字符串。

适用于大量字符串的插入以及查找。

模板题

Trie字符统计

维护一个字符串集合，支持两种操作：

  1. “I x”向集合中插入一个字符串x；

共有N个操作，输入的字符串总长度不超过 105，字符串仅包含小写英文字母。
